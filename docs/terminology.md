# CoStruct Terminology

This glossary defines the preferred Chinese-English terminology used in CoStruct.

## Core Terms

| 中文 | English | Preferred Usage |
| --- | --- | --- |
| 状态 | `state` | The collaboration phase recognized before tasking begins. |
| 协作模式 | `collaboration mode` | The interaction pattern that follows from a recognized `state`. |
| 角色 | `role` | The responsibility shape AI takes within a given `collaboration mode`. |
| 干预深度 | `intervention depth` | How deeply AI is allowed to participate in a given collaboration round. |
| 验证 | `validation` | The mechanism for checking results and triggering correction. |
| 沉淀 | `accumulation` | The mechanism for turning validated outcomes into reusable assets. |
| 资产 | `asset` | A reusable output extracted from collaboration. |
| 协作结构 | `collaboration structure` | The higher-level system CoStruct is trying to build. |

## Supporting Terms

| 中文 | English | Preferred Usage |
| --- | --- | --- |
| 文档优先 | `documentation-first` | Preferred for describing the repository nature. In Chinese prose, use `文档优先（documentation-first）` on first mention. |
| `prompt` | `prompt` | Keep lowercase when used as a general term such as `prompt` template or `prompt` repository. |
| `agent` | `agent` | Keep lowercase as a reusable system term. |
| `skill` | `skill` | Keep lowercase as a reusable system term. |
| 方法论源仓库 | `methodology source repository` | The repository where CoStruct source documents are maintained. |
| 发布仓库 | `publishing repository` | The repository used to publish public-facing pages and posts. |

## Writing Rule

- When a term already has a preferred English form above, avoid introducing a second English variant unless the meaning is intentionally different.
- In Chinese documents, use Chinese as the default surface form and keep the English term only when it improves precision or traceability.
- In English documents, use the English preferred term directly and avoid translating back into a second near-synonym.
